# APA 7th Edition Citation Guide

**CSCI 101 – APA 7th Edition Citation Guide**

<div class="step">

**Learning Objectives:** After reviewing this guide, you should be able to:

- Format narrative and parenthetical in-text citations correctly
- Construct APA 7th Edition reference entries for common source types (websites, books, journals, software)
- Apply APA formatting rules for DOIs, URLs, and multiple authors

</div>

All proposals, reports, and documentation must use APA 7th Edition format for citations and references. Below are examples of common source types in computer science projects.

------------------------------------------------------------------------

## In-Text Citations

<div class="warning">

**Two styles of in-text citation:**

- **Narrative citation:** The author's name is part of the sentence, e.g., "According to Fowler (2018), ..."
- **Parenthetical citation:** The author and year appear in parentheses at the end, e.g., "(Banks & Porcello, 2020)."

</div>

<div class="step">

### Single Author

> According to Fowler (2018), refactoring improves code maintainability without changing external behavior.

</div>

<div class="step">

### Two Authors

> React's component-based architecture promotes reusability (Banks & Porcello, 2020).

</div>

<div class="step">

### Three or More Authors

> Modern authentication systems should implement multi-factor authentication (Smith et al., 2021).

</div>

<div class="step">

### Organization as Author

> The OWASP Foundation (2023) identifies injection attacks as a top security vulnerability.

</div>

<div class="step">

### Direct Quote

> "PostgreSQL provides robust support for complex queries and data integrity" (PostgreSQL Global Development Group, 2024, para. 3).

**Block quotes (40+ words):** Indent the entire quote, omit quotation marks, and place the citation after the final period.

</div>

<div class="step">

### No Author

> Security best practices recommend rotating API keys regularly ("API Key Management," 2023).

When no author is listed, use the first few words of the title in quotation marks (for articles) or italics (for books/reports).

</div>

<div class="step">

### No Date

> The documentation recommends using environment variables for configuration (Docker, n.d.).

Use **(n.d.)** when no publication date is available.

</div>

## Formatting Your References Page

<div class="step">

- Title the page **References** (centered, bold)
- Double-space all entries
- Use a **hanging indent**: first line flush left, subsequent lines indented 0.5 inches
- Alphabetize entries by the first author's last name
- End every entry with a period (except when it ends with a URL or DOI)

</div>

------------------------------------------------------------------------

## Common Mistakes to Avoid

<div class="warning">

- **Using "Works Cited" or "Bibliography":** APA uses "References"
- **Capitalizing all words in article titles:** only capitalize the first word, first word after a colon, and proper nouns
- **Forgetting the period at the end** of a reference entry
- **Using the wrong date:** use the publication year, not the date you accessed the source (unless no date is available)
- **Missing the hanging indent:** first line flush left, continuation lines indented
- **Not italicizing titles** of books, journals, reports, and standalone works

</div>

## Tips for Technical Citations

<div class="success">

**Tip:**

- Official documentation is a valid and often preferred source for technology-specific information.
- Include version numbers when citing software or documentation that changes between versions.
- Use retrieval dates only for sources that change frequently and lack version numbers.
- DOIs (permanent links) are preferred over URLs when available.
- For comparison claims, cite benchmarks or studies that support your assertions.
- When in doubt, consult the [APA Style website](https://apastyle.apa.org/) or Freel Library resources.
