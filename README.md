# CSCI 101 — Ethics in Technology Project

This is **your** repository for the semester-long Ethics in Technology writing
project. You created it from the course template, and you will submit all six
stages of the project here as Pull Requests.

## Start here

1. Read **[SUBMISSION-RECIPE.md](SUBMISSION-RECIPE.md)** — how to submit every
   stage (the same seven steps each time).
2. Read **[Ethics_in_Technology.md](Ethics_in_Technology.md)** — the full
   project: stages, deadlines, deliverables, and grading.
3. Do **PR 00** (the practice run) in Week 1 so your first real submission
   isn't your first-ever Pull Request. The full walkthrough is in
   **[GIT-SETUP-TUTORIAL.md](GIT-SETUP-TUTORIAL.md)**.

## How submission works here

- Each stage = one Pull Request **in this repo** (base `main`, compare your
  `stage#` branch). No forks, no upstream remote, no shared branch.
- Add your instructor as a collaborator so they can review your Pull Requests
  (your repo → Settings → Collaborators).
- A friendly automated check comments on each PR to confirm your branch and
  folder are named correctly. It is advisory only — it never blocks your work,
  and **you are not graded on Git mechanics.**
